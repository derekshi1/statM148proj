## Overview
This document defines the columns generated by the journey truncation script. The script takes a full sequence of user events and "freezes" it at a random point in time to create a training snapshot for predictive modeling.

---

### Core Identifiers & Metadata
| Column Name | Description |
|:---|:---|
| `id` | The unique identifier for the specific journey (user/session ID). |
| `cutoff_time` | The randomly selected timestamp where the journey was truncated. Events occurring after this time are hidden from the model. |
| `journey_terminal_time` | The actual end time of the journey (either the moment of success or the 60-day inactivity mark). |
| `journey_terminal_reason` | Indicates why the journey ended: `success` (hit ID 28) or `inactive_60d` (churned/timed out). |

### Labels & Sampling
| Column Name | Description |
|:---|:---|
| **`label_order_shipped`** | **The Target Variable.** Binary flag (1 if the journey eventually succeeded, 0 otherwise). |
| `sampling_weight_days` | The total duration of the journey in days. Used as an oversampling weight to account for the fact that longer journeys had a higher probability of being sampled. |

### Observed Behavior (Pre-Cutoff)
These features are calculated using only events that occurred **before or at** the `cutoff_time`.

| Column Name | Description |
|:---|:---|
| `n_events_observed` | Total count of events that occurred before the cutoff. |
| `n_unique_event_types_observed` | Number of distinct event types (ED IDs) seen before the cutoff. |
| `observed_duration_days` | Time elapsed between the very first event and the cutoff time (in days). |
| `days_since_last_event` | Recency: Time elapsed between the most recent event and the cutoff time (in days). |
| `max_milestone_seen` | The highest numeric milestone value reached by the user before the cutoff. |

### Temporal Features
| Column Name | Description |
|:---|:---|
| `cutoff_hour_utc` | The hour of the day (0-23) when the snapshot was taken. |
| `cutoff_dayofweek_utc` | The day of the week (0=Monday, 6=Sunday). |
| `cutoff_month_utc` | The month of the year (1-12). |
| `cutoff_dayofyear_utc` | The day of the year (1-366). |

### State Features (First & Last)
| Column Name | Description |
|:---|:---|
| `first_ed_id` / `last_ed_id` | The Event Definition ID of the first and most recent event observed. |
| `first_event_name` / `last_event_name` | The human-readable name of the first and most recent event. |
| `first_stage` / `last_stage` | The journey stage (e.g., Discovery, Checkout) of the first and most recent event. |

### Event-Specific Counters (Sparse Features)
*Note: These columns repeat for every `ed_id` found in the Event Definitions.*

| Column Pattern | Description |
|:---|:---|
| `count_ed_[ID]` | The total number of times event ID occurred before the cutoff. |
| `seen_ed_[ID]` | Boolean flag: True if event ID occurred at least once before the cutoff. |

---

> **Note for Modeling:** To avoid **Data Leakage**, ensure that the model only uses features derived from the "Observed" and "Temporal" sections. Using `journey_terminal_reason` or `journey_terminal_time` as predictors will result in a perfect but useless model.
